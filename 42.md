## Objetivo
Implementar una regresión lineal simple con una sola feature, el kilometraje, para predecir el precio de un coche usando descenso de gradiente.

El proyecto pide dos programas:
- un predictor que estime el precio a partir de `theta0` y `theta1`
- un trainer que lea el dataset, entrene el modelo y guarde `theta0` y `theta1`

## Mandatory real
- Usar la hipótesis: `estimatePrice(mileage) = theta0 + theta1 * mileage`
- Empezar con `theta0 = 0` y `theta1 = 0`
- Entrenar con gradient descent
- Actualizar `theta0` y `theta1` simultáneamente
- Guardar los thetas para reutilizarlos en el predictor
- No usar librerías que hagan el trabajo por ti

## Masterclass, algoritmo y lógica en detalle

### 1. Qué problema intenta resolver este proyecto
Este proyecto quiere que aprendas la idea más básica de machine learning supervisado: usar ejemplos ya conocidos para construir una regla que permita hacer predicciones sobre datos nuevos.

En este caso, los ejemplos son coches de segunda mano con dos datos:
- kilometraje (`mileage`)
- precio (`price`)

La intuición humana ya te dice algo importante: normalmente, cuanto más kilometraje tiene un coche, menos vale. No siempre será perfecto, pero la relación general suele existir.

Lo que hacemos aquí es intentar que el ordenador aprenda una regla aproximada del tipo:

`precio ≈ theta0 + theta1 * kilometraje`

Esa regla es una recta.

Por tanto, el problema real del proyecto es:
- encontrar qué recta representa razonablemente bien la relación entre kilometraje y precio
- usar esa recta para estimar precios nuevos

No estamos intentando encontrar la verdad absoluta del mercado de coches.
Estamos intentando construir un modelo simple, entendible y suficientemente bueno para aproximar una tendencia.

---

### 2. Qué significa “regresión lineal”
La palabra “regresión” aquí no significa retroceder. Significa estimar un valor numérico continuo.

Ejemplos de regresión:
- predecir precio de una casa
- predecir temperatura
- predecir consumo eléctrico
- predecir salario

La palabra “lineal” significa que el modelo usa una relación lineal entre entrada y salida.

En este proyecto:
- entrada = kilometraje
- salida = precio
- modelo = una recta

La fórmula es:

`estimatePrice(mileage) = theta0 + theta1 * mileage`

Esto es exactamente la ecuación de una recta.

Si piensas en la forma clásica `y = a + bx`:
- `y` sería el precio
- `x` sería el kilometraje
- `a` sería `theta0`
- `b` sería `theta1`

---

### 3. Qué son theta0 y theta1
Estas dos variables son el corazón del proyecto.

#### theta0
Es la ordenada en el origen, o dicho de forma más práctica:
- el valor base de la recta
- el precio estimado cuando el kilometraje es 0

En el mundo real, un coche con 0 km no siempre tendrá exactamente ese precio, porque el modelo es una simplificación. Pero matemáticamente esa es su interpretación.

#### theta1
Es la pendiente de la recta.

Te dice cuánto cambia el precio cuando aumenta el kilometraje en una unidad.

Si `theta1` es negativo:
- más kilometraje -> menor precio

Si `theta1` fuera positivo:
- más kilometraje -> mayor precio

Eso en este caso probablemente no tendría sentido, así que el modelo debería tender a una pendiente negativa.

#### Ejemplo simple
Si tuvieras:
- `theta0 = 20000`
- `theta1 = -0.05`

Entonces:
- coche con 0 km -> `20000`
- coche con 100000 km -> `20000 - 0.05 * 100000 = 15000`

No es magia. Solo una recta.

---

### 4. Qué significa “entrenar” el modelo
Entrenar no es más que ajustar `theta0` y `theta1` hasta que la recta encaje razonablemente bien con los datos reales.

Al principio el enunciado te dice que arranques con:
- `theta0 = 0`
- `theta1 = 0`

Eso significa que al principio tu modelo dice:

`precio estimado = 0`

para cualquier kilometraje.

Eso es un modelo malísimo, pero sirve como punto de partida.

Luego el entrenamiento va corrigiendo esos valores poco a poco.

La idea es:
1. haces predicciones con la recta actual
2. comparas con los precios reales
3. mides cuánto te equivocas
4. corriges la recta un poquito
5. repites muchas veces

Ese “corregir un poquito” repetido muchas veces es precisamente el descenso de gradiente.

---

### 5. Qué son los datos de entrenamiento
Tienes una lista de ejemplos.
Cada ejemplo tiene:
- un `mileage`
- un `price`

Por ejemplo:
- coche A -> 24000 km, 18000 euros
- coche B -> 120000 km, 9500 euros
- coche C -> 60000 km, 13000 euros

Cada fila es una evidencia del mundo real.

El modelo no inventa conocimiento desde cero. Aprende de esos ejemplos.

Esto es aprendizaje supervisado porque para cada entrada ya conoces la salida correcta.

Sabes:
- entrada = kilometraje
- etiqueta correcta = precio

---

### 6. Qué significa “predecir”
Una vez tienes `theta0` y `theta1`, hacer una predicción es trivial.

Solo aplicas:

`estimatePrice(mileage) = theta0 + theta1 * mileage`

Por eso el proyecto está dividido en dos programas:
- uno que entrena
- otro que predice

El primero hace el trabajo difícil.
El segundo simplemente reutiliza el resultado.

Esto también refleja una arquitectura muy real:
- fase offline de entrenamiento
- fase online de inferencia/predicción

---

### 7. El concepto de error
El modelo predice un precio.
El dataset te da el precio real.

La diferencia entre ambos es el error.

Para un ejemplo concreto:

`error = precio_estimado - precio_real`

Si el error es:
- positivo -> estás estimando por encima
- negativo -> estás estimando por debajo
- cercano a 0 -> vas bien

La clave del aprendizaje es usar ese error para ajustar el modelo.

---

### 8. Por qué no basta con un solo ejemplo
Con un ejemplo aislado no puedes aprender bien una relación general.
Necesitas muchos ejemplos porque cada uno aporta información.

Si solo vieras un coche, podrías ajustar una recta que pasara por ese punto, pero no sabrías si representa la tendencia general.

Con muchos puntos, intentas encontrar una recta que capture el patrón global.

Eso es importante: la regresión lineal no intenta pasar por todos los puntos exactamente.
Intenta encontrar una recta que sea una buena aproximación global.

---

### 9. Qué es gradient descent en lenguaje simple
Gradient descent es un método iterativo para minimizar el error.

No intenta adivinar directamente la mejor recta.
Hace algo más humilde y más robusto:
- empieza con una recta mala
- mira en qué dirección mejorar
- da un pequeño paso
- repite muchas veces

Imagínate una montaña invertida o un valle.
Quieres llegar al punto más bajo.
No sabes dónde está, pero sí puedes notar hacia dónde baja el terreno.
Entonces:
- das un paso pequeño hacia abajo
- vuelves a observar
- repites

Ese proceso es gradient descent.

En este proyecto, el “terreno” es la función de error, y el punto más bajo es el conjunto de valores de `theta0` y `theta1` que mejor ajusta la recta.

---

### 10. Por qué las fórmulas del enunciado tienen esa forma
El subject te da directamente las fórmulas de actualización.
No necesitas derivarlas desde cero para pasar el proyecto, pero sí conviene entender qué están haciendo.

Las fórmulas calculan cuánto deberías corregir `theta0` y `theta1` según el error promedio en todos los ejemplos.

#### tmpTheta0
Recoge el error medio.

Si en promedio estás prediciendo demasiado alto, una corrección irá en una dirección.
Si estás prediciendo demasiado bajo, irá en la otra.

#### tmpTheta1
Recoge el error medio ponderado por el kilometraje.

Eso ajusta la pendiente.

Porque no es lo mismo equivocarte en coches con poco kilometraje que equivocarte mucho cuando el kilometraje crece. Ese término hace que la recta aprenda cómo cambia el precio según la variable de entrada.

---

### 11. Qué significa `m`
El subject hasta bromea con eso.

`m` es simplemente el número de ejemplos del dataset.

Si tienes 24 coches, entonces:
- `m = 24`

Se usa para hacer media y no suma bruta.
Eso hace que el tamaño del dataset no descontrole las actualizaciones.

---

### 12. Por qué la actualización debe ser simultánea
Esto es muy importante y en la defensa te lo pueden preguntar.

El enunciado insiste en que no olvides actualizar `theta0` y `theta1` simultáneamente.

¿Por qué?
Porque ambas correcciones deben calcularse usando el mismo estado anterior del modelo.

#### Lo correcto
1. calculas `tmpTheta0` con los theta actuales
2. calculas `tmpTheta1` con los theta actuales
3. actualizas `theta0`
4. actualizas `theta1`

#### Lo incorrecto
1. calculas `tmpTheta0`
2. actualizas `theta0`
3. calculas `tmpTheta1` usando ya el nuevo `theta0`

Eso mezcla dos estados del modelo y rompe la lógica matemática del paso de gradient descent.

Piensa que cada iteración debería ser un paso coherente sobre el mismo punto del espacio de parámetros.

---

### 13. Qué es el learning rate
El learning rate es el tamaño del paso que das en cada iteración.

Si es demasiado grande:
- puedes pasarte
- oscilar
- divergir
- no converger nunca

Si es demasiado pequeño:
- avanzas lentísimo
- parece que no aprende
- necesitas muchísimas iteraciones

El learning rate es una especie de sensibilidad del proceso de aprendizaje.

#### Analogía
Si bajas una montaña:
- pasos gigantes -> te puedes caer o saltarte el valle
- pasos minúsculos -> tardas una eternidad

El objetivo es un tamaño razonable.

---

### 14. Qué significa converger
Converger significa que las actualizaciones cada vez son menores y el modelo se va estabilizando alrededor de una solución razonable.

Se nota porque:
- los theta dejan de cambiar mucho
- el error deja de mejorar apenas
- la recta parece ya estable

Si no converge, suele haber varios motivos:
- learning rate demasiado grande
- datos mal escalados
- bug en las fórmulas
- actualización no simultánea

### 16. Qué limitaciones tiene este modelo
Esto es importante para hablar como alguien que entiende el proyecto y no solo lo ha picado.

#### Solo usa una feature
Solo miras kilometraje.
En la realidad, el precio también depende de:
- marca
- modelo
- año
- estado
- combustible
- mantenimiento
- accidentes
- extras

#### Asume relación lineal
Puede que el precio no baje de forma perfectamente lineal.
A veces la depreciación real tiene curvas, tramos o patrones más complejos.

#### Sensible a outliers
Si tienes coches raros o precios extraños, la recta puede distorsionarse.

#### No entiende contexto
Solo ajusta números.
No entiende por qué un coche vale más o menos.

Esto no hace malo al modelo. Solo lo hace simple.
Y precisamente esa simplicidad es ideal para aprender.
